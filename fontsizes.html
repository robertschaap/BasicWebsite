Font Sizes: Comparing Pixels and Relative Units

Home Font Sizes Font Weights ☰

PX

Pixels provide an absolute value. Most browsers are set to 16px for the normal font size. If the user increases the font-size, the PX value will not change in certain browsers.

EM

Ephemeral units provide a value based on the browser's base size. 1em in this case equals 16px. If the user increases the font size, all fonts will change relative to the new size. EM fonts also inherit size from other elements.

REM

Relative Ephemeral units behave much like EM units with the exception that they are always relative to the browser's base size.

The pixel values are absolute and are retained properly. In the EM Units section however, something strange is happening. The top field is fine, the second is 2 times the size of that, however despite us setting the last field (which is nested within the second field) to 1em, it is equal in size to the one above. That's because it has inherited it's size. The REM section again is fine because it scales according to the brower's default size. If you change your browsers default font size, you will see this effect scale.

Finally, when it comes to layout, text elements seem to be interpreted slightly different between browsers. There are browsers where increasing the default text size will not change the pixel values but will change both EM and REM. This seems to make relative units a safer choice. Also, using EM values for the layout of your page might cause inheritance issues, whereas pixels will stay the same for layout. These factors are important to consider.
